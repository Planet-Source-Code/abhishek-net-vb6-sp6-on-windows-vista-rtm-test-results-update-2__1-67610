VB6 SP6 Test on Windows Vista RTM Build 6000

1. Installation:

When you try to run the Visual Studio setup, you will revive a warning that Visual C++ is not compatible; just ignore the warning since we want to install VB6

  1. Install VB as usual
  2. After installed run VB6 as an administrator by right clicking the VB6 shortcut and selecting "Run as Administrator" this will completely sets up the registry entries for VB6 and its controls
  3. Install Service Packs and repeat the step 2

3. Known Issues:

  1. Redraw problem when dragging & resizing controls: You will face redraw problem when dragging the controls or resizing them but only if you are using Windows Aero interface, it works fine under "Windows Vista Basic" and "Windows Standard"
    Solution: change windows theme to "Windows Vista Basic" or "Windows Standard"

  2. Regsvr32 fails to register controls: regsvr32 will fail to register control if user account control is turned on, its turned on by default
    Solution: turn off user account protection by going to control panel > user accounts > Turn User Account Control On or off > select uncheck and reboot computer

4. Running VB6 Correctly:

To correctly run VB6, run it as an administrator, even though you account type is administrator the application are lunched as basic users only, so you need to run VB6 as with full rights.

  1. Option 1 (Best Option): Turn off user account protection by going to Control Panel > User Accounts > Turn User Account Control On or Off > select uncheck and reboot
  2. Option 2: Create a VB6 shortcut to desktop > right click > advanced > check 'run as administrator' and use this shortcut to open VB6 every time you need.

5. The Code Sample:

Please see the code sample IsAdmin, this code allows you to check whether the app is running with admin right or not, try running it with user account protection turned on, it will say, user is not admin, but if you run it by right clicking > 'run as administrator' it will say user is admin. by turning off user account protection, all applications are run as administrator.
